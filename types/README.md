# TypeScript Types

This directory contains TypeScript type definitions and interfaces used throughout the FIRST Global Team Kenya application.

## Structure

```
types/
├── api.ts              # API response and request types
├── application.ts      # Application form types
├── auth.ts             # Authentication types
├── index.ts            # Main type exports
└── prisma.ts           # Database model types (generated)
```

## API Types (`api.ts`)

### Response Types
```typescript
export interface ApiResponse<T = any> {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
    details?: any;
  };
  pagination?: {
    page: number;
    limit: number;
    total: number;
    pages: number;
  };
}

export interface PaginatedResponse<T> extends ApiResponse<T[]> {
  pagination: {
    page: number;
    limit: number;
    total: number;
    pages: number;
  };
}
```

## Utility Types

### Common Utility Types
```typescript
export type Nullable<T> = T | null;
export type Optional<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;
export type RequiredFields<T, K extends keyof T> = T & Required<Pick<T, K>>;

export type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};
```

## Best Practices

### Type Organization
- Group related types in feature-specific files
- Use descriptive, specific names
- Avoid generic names like `Data` or `Info`
- Export all types for external use

### Type Safety
- Use strict TypeScript settings
- Avoid `any` type except in utilities
- Implement proper error types
- Validate API responses with types

### Documentation
- Add JSDoc comments for complex types
- Document type parameters and constraints
- Include usage examples in comments
- Keep types synchronized with implementation

### Maintenance
- Update types when API changes
- Remove unused type definitions
- Refactor types for reusability
- Version type changes appropriately

## Usage Guidelines

### Importing Types
```typescript
// Import specific types
import type { User, Application } from '@/types';

// Import all types from a module
import * as ApiTypes from '@/types/api';

// Use in components
interface UserCardProps {
  user: User;
  onEdit: (user: User) => void;
}
```

### Type Guards
```typescript
export function isUser(obj: any): obj is User {
  return obj && typeof obj.id === 'string' && typeof obj.email === 'string';
}

export function isApplicationStatus(status: string): status is ApplicationStatus {
  return ['draft', 'submitted', 'under_review', 'accepted', 'rejected'].includes(status);
}
```

### Generic Constraints
```typescript
export interface ApiListResponse<T extends { id: string }> {
  items: T[];
  total: number;
  hasMore: boolean;
}

// Usage
type UserListResponse = ApiListResponse<User>;
type ApplicationListResponse = ApiListResponse<Application>;
```

This type system ensures type safety, better developer experience, and maintainable code throughout the application.
